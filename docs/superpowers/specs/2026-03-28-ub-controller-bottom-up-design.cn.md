# 设计规范：UB控制器自底向上设计（PCS & DLL层）

**日期：** 2026-03-28
**状态：** 待审核草案
**目标：** UnifiedBus (UB) 基础规范版本2.0（PHY模式-2）

## 1. 概述

本文档规定了UB控制器的自底向上实现设计，重点关注物理编码子层（PCS）和数据链路层（DLL）。目标配置为PHY模式-2（53G/106G PAM4），支持4或8通道。

## 2. 架构

### 2.1 数据链路层（DLL）

DLL使用基于flit的传输提供可靠的点对点数据包传输。

*   **帧格式：**
    *   **DLLDP（数据包）：** 1-512个flit
    *   **DLLCB（控制块）：** 1-32个flit
    *   **DLLDB（数据块）：** 每个段最多32个flit
*   **头部：**
    *   **LPH（链路数据包头）：** 4字节（第一个DLLDB的第一个flit）
    *   **LBH（链路块头）：** 2字节（中间/最后DLLDB的第一个flit）
*   **可靠性：**
    *   **BCRC（块CRC）：** 每个DLLDB末尾的CRC-32（4字节）
    *   **重传缓冲区：** 等待ACK的flit的TX存储区
    *   **流量控制：** 基于信用，每个虚拟通道（最多16个VL）
*   **内部接口：** 来自网络层的简单Ready/Valid基于flit的接口
    *   `flit_data[159:0]`：20字节flit有效载荷
    *   `flit_valid`：表示总线上有效数据
    *   `flit_ready`：来自DLL（TX）或网络层（RX）的背压
    *   `flit_sop`：数据包起始（DLLDP的第一个flit）
    *   `flit_eop`：数据包结束（DLLDP的最后一个flit）
    *   `flit_vl[3:0]`：数据包的虚拟通道ID

### 2.2 物理编码子层（PCS）

PCS管理FEC、加扰和通道分配。

*   **加扰器：** 基于PRBS的随机化以防止数据模式
*   **FEC：** GF(2^8)中的RS(128,120,T=4) Reed-Solomon编码/解码
    *   实现：用于高吞吐量的并行系统架构
*   **Gray编码器：** 将位映射到PAM4调制符号
*   **通道分配器：** 符号跨物理通道（4或8）条带化
*   **对齐标记（AMCTL）：** 插入标记以进行多通道去歪斜

### 2.3 控制和状态逻辑（LMSM）

*   **状态：** `Disabled`、`Param_Init`、`Credit_Init`、`Normal`
*   **CSR：** 端口配置（通道、速度、FEC）、状态监控和错误计数器

## 3. 数据流

### 3.1 发送（TX）路径

1.  **网络层**通过Ready/Valid基于flit的接口发送数据包
2.  **DLL**将数据包分段为DLLDB（最多32个flit），添加LPH/LBH和BCRC
3.  **DLL**将flit存储在重传缓冲区
4.  **PCS**对flit流进行加扰
5.  **PCS**应用RS-FEC编码
6.  **PCS**对位进行Gray编码为符号
7.  **PCS**将符号分配到通道并插入AMCTL标记
8.  **PMA接口**将符号流发送到SerDes

### 3.2 接收（RX）路径

1.  **PMA接口**从SerDes接收符号
2.  **PCS**检测AMCTL并对通道去歪斜
3.  **PCS**对符号进行Gray解码回位
4.  **PCS**执行RS-FEC解码（纠错）
5.  **PCS**对位流进行解扰
6.  **DLL**检查每个DLLDB的BCRC
7.  **DLL**解析LPH/LBH并重组数据包
8.  **DLL**通过Ready/Valid基于flit的接口将完成的数据包转发到**网络层**

## 4. 资源和时序目标

*   **时钟频率：** 目标250MHz - 400MHz，取决于内部总线宽度（128-512位）
*   **吞吐量：** 必须支持100Gbps+有效线路速率
*   **延迟：** 通过FEC/CRC路径的最小延迟

## 5. 测试策略

*   **模块级：** FEC、CRC和加扰器的UVM/Verilog测试平台
*   **集成：** TX和RX路径之间的回环测试
*   **协议一致性：** 对照UB基础规范2.0数据包格式验证
